# Task 2 – API Integration

The program uses the **DummyJSON API** to fetch product data.

### Step 1 – Fetch Products
The program sends a **GET request** to retrieve 20 products and displays:

- Product ID
- Title
- Category
- Price
- Rating

### Step 2 – Filter and Sort
Products are filtered where:
rating >= 4.5


Then the filtered list is **sorted by price in descending order**.

### Step 3 – Category Search
The program fetches products from the **laptops category** and prints:

- Laptop name
- Price

# Task 3 – Exception Handling

This section ensures the program handles errors properly.

### Safe Divide Function
A function `safe_divide()` is created to handle:

- Division by zero
- Invalid input types

Example: safe_divide(10,2)
safe_divide(10,0)
safe_divide("ten",2)


### Safe File Reader
A function `read_file_safe()` reads a file safely and handles:

- FileNotFoundError

### Robust API Calls
API calls are wrapped inside **try–except blocks** to handle:

- Connection errors
- Timeouts
- Unexpected errors

### Input Validation Loop
The program repeatedly asks the user for a **product ID (1–100)**.

The loop ensures:
- Only numbers are accepted
- The number is within the valid range
- The user can exit using **"quit"**

# Task 4 – Error Logger

An error logging system is implemented using a function called:
log_error(location, message)


Errors are written to **error_log.txt** with:

- Timestamp
- Error location
- Error message

Example log entry:
[2026-04-01 14:32:10] ERROR in fetch_products: ConnectionError — No connection could be made


The program intentionally triggers errors to demonstrate logging.
