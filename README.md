# Python Assignment — Part 3

## File I/O, APIs & Exception Handling

### Theme: Product Explorer & Error-Resilient Logger

This project demonstrates the use of **file handling, API integration, exception handling, and logging** in Python.
The program interacts with a public API to retrieve product data, processes it, saves information to files, and handles errors gracefully.

The API used in this project is **DummyJSON**, and HTTP requests are made using the **Requests** library.

# Task Overview

## Task 1 — File Read & Write Basics

### Part A — Write

The program:

* Creates a file called **python_notes.txt**
* Writes 5 Python learning notes into the file
* Appends 2 additional notes
* Displays confirmation messages after writing and appending

### Part B — Read

The program:

* Reads the contents of **python_notes.txt**
* Prints each line with numbering
* Displays the total number of lines
* Allows the user to search the file using a keyword (case-insensitive)

Example output:

```
1. Topic 1: Variables store data. Python is dynamically typed.
2. Topic 2: Lists are ordered and mutable.
...
Total number of lines: 7
```

# Task 2 — API Integration

The program retrieves product data from the **DummyJSON Products API**.

### Step 1 — Fetch Products

Fetches 20 products from:

```
https://dummyjson.com/products?limit=20
```

Displays them in a formatted table containing:

* Product ID
* Title
* Category
* Price
* Rating

---

### Step 2 — Filter and Sort

From the fetched products:

* Filters products with **rating ≥ 4.5**
* Sorts them by **price in descending order**

---

### Step 3 — Search by Category

Fetches all **laptop products** using:

```
https://dummyjson.com/products/category/laptops
```

Displays:

* Laptop name
* Price

---

### Step 4 — POST Request

Sends a simulated POST request to create a product:

```
https://dummyjson.com/products/add
```

Example data sent:

```
{
"title": "My Custom Product",
"price": 999,
"category": "electronics",
"description": "A product I created via API"
}
```

The API returns a response with a generated product ID.

# Task 3 — Exception Handling

## Part A — Guarded Calculator

A function **safe_divide(a, b)** is implemented to safely perform division.

Handles:

* `ZeroDivisionError`
* `TypeError`

Example:

```
safe_divide(10,2)
safe_divide(10,0)
safe_divide("ten",2)
```

---

## Part B — Guarded File Reader

Function **read_file_safe(filename)**:

* Reads a file safely
* Handles **FileNotFoundError**
* Uses a **finally block** to confirm file operation attempt

Tested with:

```
python_notes.txt
ghost_file.txt
```

---

## Part C — Robust API Calls

All API requests are wrapped in try-except blocks to handle:

* ConnectionError
* Timeout
* Unexpected errors

Timeout is implemented using:

```
requests.get(url, timeout=5)
```

---

## Part D — Input Validation Loop

The program continuously asks the user to enter a **product ID (1–100)**.

Features:

* Validates input before making an API request
* Prevents invalid API calls
* Displays product **title and price** if found
* Displays **"Product not found"** for invalid IDs
* Allows exit using the command **quit**

Example prompt:

```
Enter a product ID to look up (1–100), or 'quit' to exit:
```

---

# Task 4 — Error Logging System

An error logging system is implemented to store errors in **error_log.txt**.

Each log entry includes:

* Timestamp
* Error type
* Function location
* Error description

Example log entry:

```
[2025-01-15 14:32:01] ERROR in fetch_products: ConnectionError — No connection could be made
```

Two errors are intentionally triggered:

1. **ConnectionError** using an invalid URL
2. **HTTP error (404)** using a non-existent product ID

After logging, the program prints the contents of **error_log.txt**.
